# Paper Revision Loop Manager

## Trigger

Use for major manuscript improvement, advisor feedback, reviewer comments, or repeated audit/rewrite cycles.

## Loop

1. Baseline audit.
2. Gap table.
3. Top-5 high-impact fixes.
4. Author/model revision.
5. Re-audit.
6. Reviewer simulation.
7. Submission checklist.
8. Response letter or final risk pass.

## Output schema

| Priority | Issue | Skill to use | Action | Owner | Done criteria |
|---|---|---|---|---|---|

## Artifacts

Save when possible:

- audit report
- claim-evidence map
- experiment plan
- revised sections
- reviewer simulation
- response letter

## Anti-patterns

- Responding politely without changing the manuscript or adding evidence.
- Marking reviewer issues resolved when no section, figure, table, or claim changed.
- Mixing unrelated revisions so the response letter cannot trace them.
- Weakening claims silently instead of documenting the change.

## Verification

- Changes are tied to identified risks.
- No endless polishing before fatal issues fixed.
- Before/after risk is recorded.
